# Sources

Authoritative documentation for git workflow best practices.

## Commit Messages

- [How to Write a Git Commit Message](https://cbea.ms/git-commit/) - The definitive guide to the seven rules
- [Git Commit Message Guidelines (GitHub Gist)](https://gist.github.com/robertpainsi/b632364184e70900af4ab688decf6f53) - Comprehensive community guidelines
- [Pro Git: Commit Guidelines](https://git-scm.com/book/en/v2/Distributed-Git-Contributing-to-a-Project#_commit_guidelines) - Official Git documentation

## Merge and Rebase

- [Atlassian: Merging vs Rebasing](https://www.atlassian.com/git/tutorials/merging-vs-rebasing) - Visual guide to the differences
- [Pro Git: Rebasing](https://git-scm.com/book/en/v2/Git-Branching-Rebasing) - Official documentation with examples
- [Graphite: Rebase vs Fast-Forward](https://graphite.com/guides/git-rebase-vs-fast-forward) - Modern workflow guide

## Workflows

- [GitHub Flow](https://docs.github.com/en/get-started/using-github/github-flow) - Lightweight branch-based workflow
- [GitLab Flow](https://about.gitlab.com/topics/version-control/what-is-gitlab-flow/) - Extends GitHub Flow with environments
- [Trunk-Based Development](https://trunkbaseddevelopment.com/) - High-frequency integration approach

## Tools

- [Git Documentation](https://git-scm.com/doc) - Official reference
- [GitHub CLI](https://cli.github.com/) - Command-line interface for GitHub
- [Graphite](https://graphite.dev/) - Stacked diffs workflow tool

## Conflict with Other Sources

This skill follows these priorities:
1. Pro Git book (official documentation)
2. cbea.ms commit guide (industry standard)
3. Atlassian tutorials (comprehensive visuals)

When sources conflict, prefer simpler, more widely-adopted practices.
